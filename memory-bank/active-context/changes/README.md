## 🔧 維護說明

### 變更記錄原則
1. **合併優先**: 相同日期的變更應合併為單一文檔
2. **清晰標題**: 使用日期 + 變更主題命名
3. **完整記錄**: 包含原因、方法、成效、驗證
4. **狀態標記**: 明確標記已實施 vs 設計中

### 檔案命名規範
```
YYYY-MM-DD-變更主題.md          # 單一變更
YYYY-MM-DD-consolidated-changes.md  # 合併變更（推薦）
```

### 定期清理
- **每週**: 合併瑣碎變更
- **每月**: 歸檔舊變更到 archive/
- **每季**: 精簡歷史記錄
